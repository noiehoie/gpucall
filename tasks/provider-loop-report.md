## Defects found and fixed in this session

### 1. Synthetic dry-run evidence expired with no maintainer (product defect)

`onboarding-ready` requires admin-automation synthetic dry-run evidence with a
3600s TTL, but only `setup apply` ever wrote it. The production gateway
degraded to `onboarding-blocked` within an hour of every setup and stayed
there (observed on the gateway host, 12 days stale). Fixed in v2.0.68: the watch
service refreshes evidence that setup produced. Verified live: <gateway-host>
returned to `onboarding-ready` immediately after upgrade.

### 2. Route validation evidence silently invalidated by config drift (product defect)

Route validation evidence is scoped to the exact config hash. A config change
on 2026-06-29 invalidated all light/vision route evidence; only the rank
standard route was re-validated (by the intake loop). Four light routes and
one vision route were silently out of production for 9 days — every
light-intent caller request failed `NO_ELIGIBLE_TUPLE` with reason
`missing route validation evidence`.

- Environment repair: all 5 routes re-validated via `gpucall tuple-smoke`
  (canary 1 → fanout 4, budget 0.16 USD per smoke). Caller canary returned to
  5/5 PASS.
- Product fix (v2.0.69): the admin watch service now re-validates
  previously-passed routes whose evidence was invalidated by config-hash or
  commit drift — at most one route per 600s window, 3600s per-route retry
  gap, only under the operator's existing
  `recipe_inbox_auto_billable_validation` consent and budget. Failed smokes
  are never auto-retried.

### 3. Catalog/worker context contract mismatch (product defect)

Surface catalog declared `max_model_len: 131072` for Modal Qwen2.5 routes
while the deployed worker capped those models at 32768. The router trusted
the catalog and dispatched an 88,049-token production rank workload to the
tuple; vLLM rejected it (`decoder prompt longer than the maximum model
length`) surfaced as an unclassified 502 `PROVIDER_ERROR`. Fixed in v2.0.69:
the worker honors the catalog through the Qwen2.5 model-card YaRN
configuration for 7B/14B/32B, caches engines by loaded context length, and
keeps the non-YaRN streaming path at the native bound.
